Writing tools should leave ordinary files behind

AI can be useful inside the writing loop, but the document itself should remain a durable artifact that survives the tool.

The promise of a local-first writing app is simple: open a folder, write in Markdown, and keep the work portable. The app can add structure, formatting, and AI assistance, but it should never make the user wonder where the real document lives.

What the first version has to prove

Before importers, embeddings, or publishing, the product has to earn trust in the basic loop. It should read and write files without surprises, preserve every heading and list on round-trip, and make each save feel predictable.⇥ Tab to accept · Esc
